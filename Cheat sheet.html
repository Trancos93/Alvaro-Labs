Setting up your environment

First, and before we start coding, we need to settup everything we need to start working. Open a terminal and type in "Code", VS Code should open. Create a folder with the name of the project. Inside it, create an HTML file called "index.html". Also, create a folder inside the project called "Resources", where we would save our images. Inside Resources, create a folder called "Fonts". Within VS Code, create a new file, called "Index.css" and save it into Resources. Open a live server of the html file.

Those would be the first steps I'd take before I started writing a single line of code.

Plan ahead

Never start coding before having a plan. Yo can have an idea, but for coding to be productive you need to have a cliear plan of how the website is going to be, how many menus are you going to create, what images are you going to use, etc...

You need to have 100% clear how the layout of the website is going to be. Once you've got that, you can start the proyect.

Divide problems into smaller and more manageable ones

When we think about styling an HTML page, creating a new site or desingning a new function, they can be daunting tasks. However, I've found that, while creating this site for example, it's been extremely useful to divide every task. For instance, creating the header would be a task, then I would focus my attention in the article/content section, I would afterwards focus on the footer, and so on.

The thing is, if you as a developer keep thinking about everything as a whole, and everything you still have to do to fully complete the proyect, it can be daunting and overwhelming. For this reason, focusing your attention on one aspect of the site at a time and dividing every piece of work into different sections, can help a whole lot in your developer journey.

Create your own templates

Templates allow us to create faster. They allow us to start working and save us some precious time from the creative part. In a template, a developer may save or store their favorite fonts, colors, or other elements so that can recall that information faster and save some time.

Personally, I created a template featuring my favorite colors to date, the only font i've been lucky to use, the one and only (I saw it in another website and it caught my attention), and that's about it.
